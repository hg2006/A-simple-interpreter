15 Q6 (posted March 2, due March 9)

Language: Racket.

In this question, you will add seq (implementing a restricted version of begin) and set (implementing set!)

to a basic faux Racket interpreter that you can find here. Make sure you understand it.

Here is the grammar for the interpreted language.

 

‹expr›

 ::= 

‹num›

 

  |  

‹var›

 

  |  

(+ ‹expr› ‹expr› )

 

  |  

(* ‹expr› ‹expr› )

 

  |  

(- ‹expr› ‹expr› )

 

  |  

(/ ‹expr› ‹expr› )

 

  |  

( ‹expr› ‹expr› )

 

  |  

(fun ( ‹var› ) ‹expr› )

 

  |  

(with (( ‹var› ‹expr› )) ‹expr› )

 

  |  

(seq ‹expr› ‹expr› )

 

  |  

(set ‹var› ‹expr› )

Here are the new AST structures.

(struct seq (fst snd) #:transparent)
(struct set (var newval) #:transparent)

The interpreter now consumes an expression, an environment, and a store, and produces a result structure, which contains both the value of the interpreted expression and the new store.

(struct result (val newstore) #:transparent)

Of course, the starter code will have to be extensively rewritten, using the Haskell implementation in the tutorial summaries as a guideline.

Test expressions will be of the forms:

(parse exp)
(result-val (interp (parse exp) empty empty))

To submit: Q6.rkt.
