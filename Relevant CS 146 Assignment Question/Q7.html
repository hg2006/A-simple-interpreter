17 Q7 (posted March 7, due March 14)

Language: Racket.

The designers of Scheme made relatively few mistakes, but arguably one of them was to include set! instead of boxes.

In this question, you will add box, unbox, and setbox (implementing set-box!) to the starter code for Q6 (note that you do not have to support the features added in Q6, except seq). Here is the grammar for the interpreted language.

 

‹expr›

 ::= 

‹num›

 

  |  

‹var›

 

  |  

(+ ‹expr› ‹expr› )

 

  |  

(* ‹expr› ‹expr› )

 

  |  

(- ‹expr› ‹expr› )

 

  |  

(/ ‹expr› ‹expr› )

 

  |  

( ‹expr› ‹expr› )

 

  |  

(fun ( ‹var› ) ‹expr› )

 

  |  

(with (( ‹var› ‹expr› )) ‹expr› )

 

  |  

(seq ‹expr› ‹expr› )

 

  |  

(box ‹expr› )

 

  |  

(unbox ‹expr› )

 

  |  

(setbox ‹expr› ‹expr› )

Here are the new AST structures.

(struct seq (exp1 exp2) #:transparent)
(struct newbox (exp) #:transparent)
(struct openbox (exp) #:transparent)
(struct setbox (bexp vexp) #:transparent)

You can use your Q6 implementation as a starting point, removing set and adding the new features. The result of interpreting a box expression should be a location.

To submit: Q7.rkt.
